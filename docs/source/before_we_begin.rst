在开始之前
======================

本节介绍了 CFMS 项目的当前状态、适用人群和技术要求。请仔细阅读以下内容，以确保您了解
项目的局限性和风险，并采取适当的应对措施。

项目状态
--------------

.. warning::

   **请谨慎在生产环境中使用 CFMS**
   
   尽管我们近来已在这方面取得了一些进展，但 CFMS 的安全性尚未得到充分验证。
   通常只推荐在满足以下条件时使用 CFMS：
   
   - 已进行全面的安全审计
   - 理解并接受所有潜在风险
   - 有能力及时修复可能出现的安全问题
   - 为潜在的数据丢失或泄露做好了准备

CFMS 尚处于 **早期开发阶段**。许多计划中的功能尚未实现，而现有功能也缺乏完善的测试。
API规范和内部实现在服务端不同版本之间可能发生重大变化。

我们尚在对已有的功能进行安全审计，并致力于改进系统的安全性和稳定性。然而，系统存在未
发现的漏洞和缺陷是极有可能的，因此不建议在审计完成前将重要数据部署到服务端中。

项目背景
---------

CFMS 最初是为视觉小说《溯流而上》的需求而开发的。在开发优先级上，功能实现和用户界面优先于安全性考量。
虽然项目实现了基本的安全措施（如密码哈希、JWT 认证、SSL/TLS 加密等），但这些措施的有效性尚未经过
严格的安全审查。

我们欢迎志愿者参与安全审计和改进工作。如果您有相关经验并愿意贡献，请通过 GitHub Issues 联系我们。

称谓说明
---------

如无特别说明，本手册仅针对 CFMS on Websocket 服务端而编写。
